This is an early draft of Minimalist Data Wrangling with Python by Marek Gagolewski. It is distributed in the hope that it will be useful. Any bug/typos reports/fixes are appreciated. Although available online, this is a whole course, and should be read from the beginning to the end. In particular, refer to the Preface for general introductory remarks.

Before we start, letâ€™s do the usual chores:

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns

Consider a small subset of the Urban Forest dataset which originally includes information on over 77,000 trees in Melbourne, VIC, Australia.

trees = pd.read_csv("https://raw.githubusercontent.com/gagolews/" +
    "teaching_data/master/marek/urban_forest_subset2.csv",
    comment="#")
trees = trees.rename({
    "Common Name": "Species",
    "Diameter Breast Height": "Diameter",
    "Year Planted": "Year",
    "Located In": "Location",
    "Age Description": "Age",
}, axis=1).loc[:, ["Species", "Location", "Age", "Diameter", "Year"]]
trees.head()
##              Species Location          Age  Diameter  Year
## 0       London Plane   Street     Juvenile        16  2009
## 1   Japanese Zelkova   Street  Semi-Mature        12  2008
## 2  Lemon Scented Gum   Street     Juvenile         8  2009
## 3   Japanese Zelkova   Street  Semi-Mature        20  2008
## 4   Japanese Zelkova   Street  Semi-Mature        18  2006

We have a mix of categorical (species, location, age) and numerical (diameter, year) variables. Unless we had encoded qualitative variables as integers, this would not be possible with plain matrices.

In this section, we will treat the qualitative columns as grouping variables, so that we can, e.g., summarise or visualise the data in each group separately, because it is likely that data distributions vary across different factor levels. This is much like having many data frames stored in one object.

trees is thus an example of heterogeneous data at their best.

15.1. The groupby MethodsÂ¶

DataFrame and Series objects are equipped with the groupby methods, which assist in performing a wide range of popular operations in data groups defined by one or more data frame columns (compare [Wic11]).

They return objects of class DataFrameGroupBy and SeriesGroupby:

type(trees.groupby("Location"))
## <class 'pandas.core.groupby.generic.DataFrameGroupBy'>
type(trees.groupby("Location")["Diameter"])  # or (...).Diameter
## <class 'pandas.core.groupby.generic.SeriesGroupBy'>

Important

DataFrameGroupBy and SeriesGroupBy inherit from (extend) the GroupBy class, hence they have many methods in common. Knowing that they are separate types is useful in exploring the list of possible methods and slots in the pandas manual.

trees.groupby("Location").size()

15.1.1. Aggregating Data in GroupsÂ¶

The DataFrameGroupBy and SeriesGroupBy classes are equipped with a number of well-known aggregation functions, for example:

trees.groupby("Location").mean().reset_index()
##   Location   Diameter         Year
## 0     Park  22.629630  2008.185185
## 1   Street  15.465753  2007.575342

Note that the arithmetic mean was computed only on numeric columns. Further, a few common aggregates are generated by describe:

trees.groupby("Location")["Diameter"].describe().reset_index()
##   Location  count       mean        std  min  25%   50%   75%    max
## 0     Park   27.0  22.629630  26.029788  2.0  4.0  18.0  26.5  114.0
## 1   Street   73.0  15.465753  11.530253  1.0  8.0  13.0  20.0   70.0

But we can always apply a custom set of functions:

(trees.loc[:, ["Age", "Diameter", "Year"]].groupby("Age")
    .aggregate([np.mean, np.median]).reset_index())
##            Age   Diameter                Year        
##                      mean median         mean  median
## 0     Juvenile   9.428571    9.0  2007.464286  2008.0
## 1       Mature  75.600000   70.0  2007.600000  2008.0
## 2          New   3.411765    3.0  2009.588235  2010.0
## 3  Semi-Mature  20.800000   20.0  2007.280000  2007.5

Note that the resultâ€™s columns slot is a hierarchical index.

Note

(*) Custom (anonymous) functions can be generated on the fly using the so-called lambda expressions:

(trees.loc[:, ["Age", "Diameter", "Year"]]
    .groupby("Age").aggregate(lambda x: (np.max(x)-np.min(x))/2)
    .reset_index())
##            Age  Diameter  Year
## 0     Juvenile       6.5   2.0
## 1       Mature      31.5   1.0
## 2          New       2.5   1.0
## 3  Semi-Mature      14.5   1.5

Note that the syntax is lambda argument_name: return_value.

Note

(**) the column names in the output object are generated by reading the applied functionsâ€™ __name__ slots, see, e.g., print(np.mean.__name__).

mr = lambda x: (np.max(x)-np.min(x))/2
mr.__name__ = "midrange"
(trees.loc[:, ["Age", "Diameter", "Year"]]
    .groupby("Age").aggregate([np.mean, mr]).reset_index())
##            Age   Diameter                  Year         
##                      mean midrange         mean midrange
## 0     Juvenile   9.428571      6.5  2007.464286      2.0
## 1       Mature  75.600000     31.5  2007.600000      1.0
## 2          New   3.411765      2.5  2009.588235      1.0
## 3  Semi-Mature  20.800000     14.5  2007.280000      1.5

15.1.2. Transforming Data in GroupsÂ¶

We can easily transform individual columns relative to different data groups by means of the transform method for GroupBy objects.

def standardise(x):
    return (x-np.mean(x, axis=0))/np.std(x, axis=0)

trees["Diameter_Species_Std"] = (trees.loc[:, ["Species", "Diameter"]]
    .groupby("Species").transform(standardise))
(trees.loc[:, ["Species", "Diameter", "Diameter_Species_Std"]]
    .groupby("Species").aggregate([np.mean, np.std]))
##                     Diameter            Diameter_Species_Std          
##                         mean        std                 mean       std
## Species                                                               
## Japanese Zelkova   11.214286   7.847421         9.516197e-17  1.018350
## Lemon Scented Gum   9.680000   6.606310         3.663736e-17  1.020621
## London Plane       21.393939  16.813201        -2.691450e-17  1.015505
## Peppercorn Tree    34.142857  26.935413        -1.110223e-16  1.037749

The new column gives the relative z-scores: a London plane tree with relative z-score of 0 has the diameter at breast height of 21.4 cm, whereas a lemon scented gum tree with the same z-score has the true diameter of 9.7 cm.

15.3. Manual Splitting Into Subgroups (**)Â¶

Let us demonstrate how easily the splitting can be done manually without the use of pandas, which in some cases can be limiting. Calling numpy.split(arr, ind) returns a list with arr (being an array-like object, e.g., a matrix, a vector, or a data frame) split rowwisely into len(ind)+1 chunks at indices given by ind.

For example:

np.split(np.arange(10)*10, [3, 7])
## [array([ 0, 10, 20]), array([30, 40, 50, 60]), array([70, 80, 90])]

To split a data frame into groups defined by a categorical column, we can first sort it with respect to the criterion of interest, for instance, the Location data:

trees_srt = trees.sort_values("Location", kind="stable")

Then, we can use numpy.unique to fetch the indices of first occurrences of each series of identical labels:

levels, where = np.unique(trees_srt.loc[:, "Location"], return_index=True)
levels, where
## (array(['Park', 'Street'], dtype=object), array([ 0, 27]))

This can now be used for dividing the sorted data frame into chunks:

trees_grp = np.split(trees_srt, where[1:])

We obtained a list of data frames split at rows specified by where[1:]. Here is a preview of the first and the last row in each chunk:

for i in range(len(levels)):
    print(f"level='{levels[i]}'; preview:")
    print(trees_grp[i].iloc[ [0, -1], : ])
    print("")
## level='Park'; preview:
##             Species Location          Age  Diameter  Year  Diameter_Species_Std
## 12  Peppercorn Tree     Park  Semi-Mature        21  2009             -0.506359
## 99  Peppercorn Tree     Park  Semi-Mature        40  2008              0.225660
## 
## level='Street'; preview:
##              Species Location          Age  Diameter  Year  Diameter_Species_Std
## 0       London Plane   Street     Juvenile        16  2009             -0.325790
## 98  Japanese Zelkova   Street  Semi-Mature        19  2008              1.010343

We can apply any operation on each subgroup we have learned so far, our imagination is the only limiting factor, e.g., aggregate the two numeric columns:

trees_agg = [
    dict(
        level=t.loc[:, "Location"].iloc[0],
        diam_mean=np.mean(t.loc[:, "Diameter"]),
        year_mean=np.mean(t.loc[:, "Year"])
    )
    for t in trees_grp
]
trees_agg
## [{'level': 'Park', 'diam_mean': 22.62962962962963, 'year_mean': 2008.1851851851852}, {'level': 'Street', 'diam_mean': 15.465753424657533, 'year_mean': 2007.5753424657535}]

Finally, the results may be combined, e.g., to form a data frame:

pd.DataFrame(trees_agg)
##     level  diam_mean    year_mean
## 0    Park  22.629630  2008.185185
## 1  Street  15.465753  2007.575342

We see that manual splitting is very powerful but quite tedious in case where we would like to perform the basic operations such as computing some basic aggregates. These scenarios are very common, no wonder why the pandas developers came up with an additional, convenience interface in the form of the pandas.DataFrame.groupby and pandas.Series.groupby methods and the DataFrameGroupBy and SeriesGroupby classes. However, it is still worth knowing the low-level way to perform splitting in case of more ambitious tasks.
